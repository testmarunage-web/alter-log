# Alter Log 開発ルール（AIとの契約書）

## プロジェクト概要
「もう一人の自分（Alter Ego）」が裏側で密かに観測日記をつける、究極の内省・自己分析アプリ「Alter Log」。
Next.js(App Router) + Prisma + Clerk + Vercel で構成されている。
※旧プロジェクト名「Hack Log」から移行済み。

## 🚨 【最重要】絶対に破ってはいけないルール（地雷マップ）

1. **デザインと世界観の死守**
   - 【テーマカラー】背景は漆黒（`#0B0E13`）、アクセントは真鍮色（`#C4A35A`）やアッシュグレー（`#8A8276`）を厳守すること。
   - 【絵文字の禁止】UI上に絵文字（😀など）は一切使用しない。アイコンは必ずシンプルなSVGを使用すること。
   - 【フォント】明朝体や教科書体は使わず、モダンなsans-serif系をデフォルトとすること。
   - スマホのPWA（フルスクリーン）動作を前提としているため、`manifest.json` や `viewport` の設定を勝手に壊さないこと。

2. **UXの原則（同期的な待機時間の排除）**
   - APIのローディング（ぐるぐる回るスピナーなど）でユーザーを待たせるUIは極力避ける。
   - 特に「Alter Log（AIの分析日記）」の生成は、ユーザーのアクション直後に同期的に行わず、非同期（数十分〜数時間後のバックグラウンド処理）で行う設計を前提とすること。

3. **Vercelのビルドを壊すな**
   - TypeScriptの型エラーやESLintの警告を残したままコミットしないこと。
   - `next.config.ts` を修正する際は、Next.jsの最新バージョンの仕様に厳密に従うこと。

4. **データベース（Prisma）の扱い**
   - ユーザーデータは Clerk の UserID を用いてマルチテナントで完全に分離すること。
   - スキーマ（`schema.prisma`）を変更した後は、必ず `npx prisma generate` と `npx prisma db push`（またはmigrate）を行うこと。
   - Vercelデプロイ用に、`package.json` の `postinstall` スクリプトで Prisma Client が生成される状態を維持すること。

5. **勝手なリファクタリングの禁止**
   - 指示された箇所以外のコード（変数名のリネーム、別コンポーネントの整理など）を「ついでに」修正しないこと。影響範囲は最小限に留める。

## 🏗️ ナビゲーションと機能定義
- アプリの主要導線は以下の5つに分割されている。
  1. ホーム (`/dashboard`)
  2. ジャーナル (`/chat?mode=journal`)：音声入力ファーストのブレインダンプ用。
  3. 壁打ち (`/chat?mode=coach`)：AIとのチャット対話用。LINE風のタイムスタンプ付き。
  4. レポート (`/archive`)：1週間の構造化された内省データ。
  5. Alter Log (`/alter-log`)：AI視点の密かな観測日記。

## 🛠 デバッグ・開発手順
- エラーが発生した際は、勝手に推測して修正するのではなく、必ずエラーログ全体を解析してから修正案を提示・実行すること。
- コードを変更した後は、ローカル環境で画面が壊れていないか確認する手順を踏むこと。
